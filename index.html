ActivityWatch setup

Checking PEM address PEM is checking whether this page is opened from the daily localhost address.
1
Open PEM from localhost For daily use with ActivityWatch, open PEM at http://127.0.0.1:8891/index.html. Direct-file mode still works for exercises, but ActivityWatch sync cannot work there.
2
Open the ActivityWatch config folder Right-click the ActivityWatch tray icon, choose Open config folder, then open the aw-server folder.
3
Edit the active server config Your tray menu shows aw-server checked. Open aw-server/aw-server.toml. Do not use the rust config unless aw-server-rust is checked instead.
4
Add PEM to allowed browser origins In the [server] section, replace this disabled line: #cors_origins = "" With this active line: cors_origins = "http://127.0.0.1:8891" A line starting with # is turned off. Leave #port = "5600" alone; 8891 is PEM's address, not ActivityWatch's port.
Complete aw-server.toml example 
[server]
#host = "localhost"
#port = "5600"
#storage = "peewee"
cors_origins = "http://127.0.0.1:8891"

[server.custom_static]

[server-testing]
#host = "localhost"
#port = "5666"
#storage = "peewee"
#cors_origins = ""

[server-testing.custom_static]
These will still fail
  • #cors_origins = "http://127.0.0.1:8891" because it is still commented out.
  • #cors_origins = "cors_origins = ..." because the copied line was pasted inside another value.
  • cors_origins = "http://127.0.0.1:8891/index.html" because ActivityWatch needs only the origin, not the page path.
  • Editing [server-testing] because PEM uses the normal [server] section.
5
Restart ActivityWatch, then refresh PEM Quit and reopen ActivityWatch so it reads the config change, then press Refresh now. PEM imports only summarized active-time data.

Storage failure tests

Storage failure tests No URL test mode active
No storage test mode active These are debugging checks for after code changes. They intentionally break saving so you or Codex can confirm the warning and recovery paths still work.
Dismiss test warning Clears only the in-memory simulated warning. URL-based test modes stay active until you return to normal mode.
All saves fail Dev check: every app-data save fails. Change a setting; the app should warn you and the change should not stick after reload.
Next save fails once Dev check: only the next app-data save fails. Try changing a setting; the warning should appear once, then later saves should work again.
Browser storage blocked Dev check: the app acts like browser storage is unavailable. It should show a strong warning and point you toward downloading JSON.
Import rollback Dev check: importing a JSON backup fails partway through. Now import a valid JSON backup; the app should restore your previous data.
Image storage full Dev check: saving an exercise image fails. Now add or import an image; the app should show a storage-full message and not save the image.
Back to normal mode Removes pem_test from the current URL and reloads this app normally.
