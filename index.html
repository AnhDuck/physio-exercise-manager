Storage failure tests

Storage failure tests No URL test mode active
No storage test mode active These are debugging checks for after code changes. They intentionally break saving so you or Codex can confirm the warning and recovery paths still work.
Dismiss test warning Clears only the in-memory simulated warning. URL-based test modes stay active until you return to normal mode.
All saves fail Dev check: every app-data save fails. Change a setting; the app should warn you and the change should not stick after reload.
Next save fails once Dev check: only the next app-data save fails. Try changing a setting; the warning should appear once, then later saves should work again.
Browser storage blocked Dev check: the app acts like browser storage is unavailable. It should show a strong warning and point you toward downloading JSON.
Import rollback Dev check: importing a JSON backup fails partway through. Now import a valid JSON backup; the app should restore your previous data.
Image storage full Dev check: saving an exercise image fails. Now add or import an image; the app should show a storage-full message and not save the image.
